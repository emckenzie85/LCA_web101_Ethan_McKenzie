Building Accessible Websites: The Domino Effect

As I delve deeper into coding I find more reasons why building accessible websites is an absolute must. It is a web developer's responsibility to create code that not only works properly from a functional standpoint but in a legal sense as well.

Recently Domino's was sued (Domino's Pizza v. Robles, 18-1539) for not supporting access for customers with disabilities. They ended up losing due to their failure to allow compatibility with a software that reads webpages known as JAWS, short for 'Job Access With Speech.' This was surely a huge blow to the company both financially and image-wise, making this a great example of what can happen if code is not written to adhere to ALL accessibility standards.

In 1990, the Americans with Disabilities Act was passed which requires stores, office buildings and other public accomodations to take the proper steps to provide access for disabled people. This includes all forms of accessibility and since computers/smart phones are being used by everybody in today's world this is something we must always implement.

With the ever-changing technological landscape we must remain current with all code in order to ensure everybody has access. You put your heart and soul into your work, take the extra steps to make sure everybody has a chance to experience it.

Coding of the Future

DeepCode, an AI tool that helps with writing better code is developing machine learning systems to review code. As the company expands language capabilities the possibility of a coding "spell-check" is becoming a reality.

When linked, DeepCode is able to check for broken code, revealing it to developers and ultimately saving time on fixes.

As codes become more complex and the total numbers of codes increase this will surely be a tool developers should learn to utilize.
